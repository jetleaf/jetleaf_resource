Creates a composite OR condition that evaluates to true if either left or right evaluates to false.

A condition that evaluates to true only if both nested conditions evaluate to false.

This can be seen as a logical NOR operation:

WhenNone(A, B) == !(A || B)

Example

final condition = WhenNone(
  WhenExpr('#user.isAdmin'),
  WhenExpr('#user.isModerator'),
);

// Applies only when neither admin nor moderator

Useful for negating multiple disjunctive conditions in resource-level logic such as caching, rate limiting, or authorization.
